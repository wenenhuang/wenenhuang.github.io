# 正则表达式匹配与高亮工具需求文档 (PRD)

## 1. 项目概述
本项目旨在开发一个简洁、高效的在线工具，帮助用户通过正则表达式从源文本中提取目标数据，并实时高亮显示匹配结果。

## 2. 用户需求
- **源数据输入**：用户能够输入或粘贴大段文本数据。
- **正则输入**：用户能够输入自定义的正则表达式。
- **结果输出**：系统自动提取匹配到的目标数据。
- **视觉反馈**：在源文本中高亮显示匹配到的部分，方便核对。
- **易用性**：界面清爽，操作直观。

## 3. 功能需求
### 3.1 核心功能
- **文本输入区**：提供一个多行文本框（Textarea）用于存放源数据。
- **正则表达式输入框**：提供一个单行输入框用于输入正则表达式。
- **实时匹配**：
    - 当用户输入或修改正则表达式时，系统应自动进行匹配。
    - 匹配模式默认为全局匹配（Global）。
- **匹配结果列表**：
    - 在界面一侧或下方显示所有匹配到的字符串列表。
    - 显示匹配到的总数量。
- **高亮显示**：
    - 在源文本区域同步高亮显示所有符合正则的部分。
    - 考虑使用叠加层或富文本编辑器的形式来实现高亮，确保不影响用户继续编辑源文本。

### 3.2 辅助功能
- **复制功能**：一键复制所有匹配到的结果。
- **清空功能**：一键清空输入的内容。
- **常用正则推荐**：可选功能，提供如邮箱、手机号、URL等常用的正则模板。

## 4. UI/UX 需求
- **布局设计**：
    - 采用响应式布局，适配不同尺寸的屏幕。
    - 页面布局应遵循“输入-操作-输出”的逻辑顺序。
    - 避免元素堆叠，保留足够的留白（Whitespace），确保界面不拥挤。
- **视觉风格**：
    - 使用现代、扁平化的设计风格。
    - 高亮颜色应醒目但不刺眼（如淡黄色或浅蓝色背景）。
    - 字体清晰，行间距适中。

## 5. 技术需求
- **前端框架**：建议使用 React, Vue 或简单的 HTML/JS/CSS（取决于项目复杂度）。
- **匹配逻辑**：利用 JavaScript 原生的 `RegExp` 对象进行处理。
- **性能优化**：对于超长文本，优化匹配算法，避免界面卡顿。

## 6. 验收标准
- [ ] 用户可以输入文本和正则。
- [ ] 匹配结果能够实时更新。
- [ ] 源文本中的匹配项有明显的高亮标记。
- [ ] 界面布局整洁，在 1920x1080 和 1366x768 分辨率下均表现良好。
